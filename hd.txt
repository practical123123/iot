* apriori() is a function from the arules package.

* By default, Apriori uses:
   * Support = 0.1
   * Confidence = 0.8
   * Minimum rule length = 1
________________


The rules are created in the form:
{Item(s) on LHS} → {Item(s) on RHS}


  



6. summary(rules)
Working:
* Displays a summary of all the association rules generated.
Summary shows:
* Total number of rules generated
* Distribution of:
   * Support
   * Confidence
   * Lift
* Rule lengths (number of items in each rule)
* Most frequent items involved in the rules
  



7. rules<-apriori(mydata,parameter=list(minlen=2, maxlen=3,supp=.5))
Working:
* Generates association rules from mydata using the Apriori algorithm with custom parameters.

Explanation of parameters:
minlen = 2 : Minimum number of items in a rule must be 2 (LHS + RHS)
maxlen = 3 : Maximum number of items in a rule must be 3
supp = 0.5 : Minimum support is 50%
  



   8. inspect(rules)
Working:
   * Displays all the generated association rules in a readable table format.
What is shown:
   * LHS (Left-Hand Side) of the rule
   * RHS (Right-Hand Side) of the rule
   * Support
   * Confidence
   * Lift

Example output format:
{Milk, Bread} => {Butter}  support=0.6  confidence=0.85  lift=1.2


Meaning of the Measures:
      * Support (0.5)
→ Rule appears in 50% of all transactions
      * Confidence
→ Probability that RHS occurs when LHS occurs
      * Lift
→ Strength of the association
         * Lift > 1 → Strong rule
         * Lift = 1 → No association
         * Lift < 1 → Weak rule

import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score, classification_report
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler


# 1. Load the Dataset
iris = load_iris()
X = iris.data
y = iris.target
target_names = iris.target_names


# 2. Split Data into Training and Testing Sets
# 70% for training, 30% for testing (test_size=0.3)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)


# 3. Feature Scaling (Recommended for distance-based algorithms like KNN)
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)


# 4. Initialize and Train the KNN Classifier
# Choose a value for k (n_neighbors). An odd number helps avoid ties in voting.
k = 5
knn = KNeighborsClassifier(n_neighbors=k)
knn.fit(X_train_scaled, y_train)


# 5. Make Predictions
y_pred = knn.predict(X_test_scaled)


# 6. Evaluate the Model
accuracy = accuracy_score(y_test, y_pred)
print(f"Accuracy with k={k}: {accuracy:.2f}")


# Print a detailed classification report
print("\nClassification Report:")
print(classification_report(y_test, y_pred, target_names=target_names))


# 7. Use the Model to Predict a New Data Point
# Example: a new data point (sepal length, sepal width, petal length, petal width)
new_point = np.array([[5.8, 2.7, 4.1, 1.0]])
# The new point must also be scaled using the same scaler object
new_point_scaled = scaler.transform(new_point)


prediction = knn.predict(new_point_scaled)
print(f"\nPrediction for new point {new_point}: {target_names[prediction[0]]}")

Hypothesis Testing


Q1)  Suppose the manufacturer claims that the mean lifetime of a light bulb is more than 10,000 hours. In a sample of 30 light bulbs, it was found that they only last 9,900 hours on average. Assume the population standard deviation is 120 hours. At .05 significance level, can we reject the claim by the manufacturer?


CODE : 


# One-sample Z-test in R


# Given values
xbar <- 9900
mu0 <- 10000
sigma <- 120
n <- 30
alpha <- 0.05


# Z-test statistic
z <- (xbar - mu0) / (sigma / sqrt(n))


# P-value (left-tailed test)
p_value <- pnorm(z)


# Critical value
z_critical <- qnorm(alpha)


# Print results
cat("Z value:", z, "\n")
cat("P-value:", p_value, "\n")
cat("Critical Z value:", z_critical, "\n")


# Decision
if (p_value < alpha) {
  cat("Decision: Reject the manufacturer's claim\n")
} else {
  cat("Decision: Fail to reject the manufacturer's claim\n")
}




Function Used:
1. sqrt()
Syntax: sqrt(x)
Explanation: sqrt() calculates the square root of a number. It is used when computing the standard error of the mean or proportion.
2. pt()
Syntax: pt(q, df)
Explanation: pt() computes the cumulative probability (area under the curve) of the Student’s t-distribution up to a given t-value q, with df degrees of freedom. In hypothesis testing, it is used to calculate the p-value. For a right-tailed (upper-tailed) test, the p-value is found by subtracting this cumulative probability from 1, i.e., 1 - pt(t_stat, df).




input:
  

Output:
  

  





Final conclusion 
At the 0.05 significance level, there is sufficient evidence to reject the manufacturer’s claim.
The data suggest that the mean lifetime of the light bulbs is less than 10,000 hours.








Q2) A stenographer claims that she can type at the rate of 120 words per minute. Can we reject her claim on the basis of 100 trails in which she demonstrates a mean of 116 words and a standard deviation of 15 words? Use 5% level of significance.


CODE : 


# Given values
x_bar <- 116
mu0 <- 120
s <- 15
n <- 100
alpha <- 0.05


# Test type
test_type <- "Two-tailed Z-test"


# Z statistic
z <- (x_bar - mu0) / (s / sqrt(n))


# P-value (two-tailed test)
p_value <- 2 * pnorm(abs(z), lower.tail = FALSE)


# Critical value
z_critical <- qnorm(1 - alpha/2)


# Output
